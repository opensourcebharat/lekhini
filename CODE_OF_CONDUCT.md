# Code of Conduct

Lekhini is a project of
[Open Source Bharat](https://opensourcebharat.org). We want
participation in this project to be welcoming, respectful, and
productive for everyone — contributors, users, and maintainers alike,
regardless of background or experience level.

## Our standard

Lekhini adopts the
[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)
as its code of conduct. The full text is available at that link and
is incorporated by reference.

In short:

- Be kind. Assume good intent. Disagree on ideas, not people.
- Keep discussion focused on the project and the technical problem
  at hand.
- Harassment, personal attacks, and discriminatory behaviour are not
  tolerated.
- Maintainers are expected to apply this standard fairly and
  consistently, including to themselves.

## Scope

This Code of Conduct applies to all project spaces — the GitHub
repository (issues, pull requests, discussions, commit messages, code
review comments), as well as any official Lekhini or Open Source
Bharat communication channels.

It also applies when an individual is officially representing the
project in public spaces.

## Reporting

If you experience or witness behaviour that violates this Code of
Conduct, please report it privately to the maintainers at
**conduct@opensourcebharat.org**.

All reports will be reviewed and investigated promptly and fairly.
Reporter identity will be kept confidential to the extent possible.

Please do not use public GitHub issues to report Code of Conduct
violations.

## Enforcement

Maintainers may take any action they deem appropriate, up to and
including a permanent ban from project spaces, for behaviour they
consider in violation of this Code of Conduct.

For the full enforcement ladder (correction → warning → temporary ban
→ permanent ban) and the criteria for each, see the
[Contributor Covenant 2.1 enforcement guidelines](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines).
